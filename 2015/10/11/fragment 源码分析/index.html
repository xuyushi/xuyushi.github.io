fragment 源码分析

@Override
    protected void onCreate(Bundle savedInstanceState) {
        mFragments.attachActivity(this, mContainer, null);
        // Old versions of the platform didn't do this!
        if (getLayoutInflater().getFactory() == null) {
            getLayoutInflater().setFactory(this);
        }
        
        super.onCreate(savedInstanceState);
        
        NonConfigurationInstances nc = (NonConfigurationInstances)
                getLastNonConfigurationInstance();
        if (nc != null) {
            mAllLoaderManagers = nc.loaders;
        }
        if (savedInstanceState != null) {
            Parcelable p = savedInstanceState.getParcelable(FRAGMENTS_TAG);
            mFragments.restoreAllState(p, nc != null ? nc.fragments : null);
        }
        mFragments.dispatchCreate();
    }

getLayoutInflater().setFactory(this);
this 的类为 Factory
其中 Factory

   public interface Factory {

实现了onCreateView接口，即可构造自己的 view。

进去addFragment

    public void addFragment(Fragment fragment, boolean moveToStateNow) {
        if (mAdded == null) {
            mAdded = new ArrayList<Fragment>();
        }
        if (DEBUG) Log.v(TAG, "add: " + fragment);
        makeActive(fragment);
        if (!fragment.mDetached) {
            if (mAdded.contains(fragment)) {
                throw new IllegalStateException("Fragment already added: " + fragment);
            }
            mAdded.add(fragment);
            fragment.mAdded = true;
            fragment.mRemoving = false;
            if (fragment.mHasMenu && fragment.mMenuVisible) {
                mNeedMenuInvalidate = true;
            }
            if (moveToStateNow) {
                moveToState(fragment);
            }
        }
    }

最后调用了moveToState(fragment);

在 initial 中对 加载了布局文件 ，代码如下 

if (f.mFromLayout) {
    // For fragments that are part of the content view

    if (f.mView != null) {
        f.mInnerView = f.mView;

if (f.mFromLayout)代表的是你这个Fragment的生成是否是从layout.xml文件中生成的

依次调用performCreateView ->onViewCreated完成 view 的加载

fragment的生成是通过Fragment的静态工厂方法帮你生成。而且生成之后将纳入mFragments的管理。那么什么是mFragments?实际上mFragments是FragmentManager对象.

fragment 状态

activity 在 onCreate 时调用
mFragments.dispatchCreate();
moveToState(Fragment.CREATED, false);
递归调用每个 fragment 的 moveToState

状态定义如下 

    static final int INITIALIZING = 0;     // Not yet created.
    static final int CREATED = 1;          // Created.
    static final int ACTIVITY_CREATED = 2; // The activity has finished its creation.
    static final int STOPPED = 3;          // Fully created, not started.
    static final int STARTED = 4;          // Created and started, not resumed.
    static final int RESUMED = 5;          // Created started and resumed.
    
    int mState = INITIALIZING;

其核心目的是与 activity 的生命周期同步

FragmentManager的每一次状态变更,都会引起其包含的Fragment的状态变更

对于f.mStatenewState可以理解为desdroy的过程
在每个 case 语句中，根据newState 和 当前 case 的 state，判断这个 case 语句是否要执行。
