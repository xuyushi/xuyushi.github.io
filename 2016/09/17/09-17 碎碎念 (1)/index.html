09-17 碎碎念

发表于 2016-09-17 | 分类于 随笔

09/17 晨停电到现在有9小时了，仿佛没有电没有网啥也干不了，背着电脑找了家小店，点了杯咖啡，码码字，回顾整理一下这个比较郁闷的 Q

最近这三个月感觉技术上的沉淀和成长没有以往的多

感觉自己已经慢慢进入了舒适区，晚上下班回来，也不想学习了，躺着发发呆 和女友视视频，印象笔记裁剪的东西越来越多，缺一直没时间整理。

这个版本的项目迭代做的心力憔悴，几乎一半的时间都花费在与 UI 和产品 低效率的沟通中。从这个版本学到的新知识点几乎为null。只有项目的收尾阶段学习了用 tracview 来寻找优化点。

比较欣慰的是老大面试带着我了，面试的过程的发现了很多自己的不足的点，java 和 android 基础不扎实，有些知识点仿佛知道，但是一旦稍微深入一点就懵逼了

从项目中能学到的东西越来越越少，每天 rss 中订阅的各种 Android 内容都比较零散，看完了不及时总结整理的话等于白看。有必要开辟一个更好的学习的途径和方式了 。目前的想法是多思考总结优秀的代码。如何实现？这么写的好处？为什么这么写？

  1. 深入 Android 源码分析
    • [x] window
    • [x] AsyncTask
    • [x] Looper、Handler、Message
    • [x] view 事件传递
    • [ ] view 绘制
    • [ ] 四大组件启动过程
    • [ ] 个大 serviceManager

  2. 学习一个完整优秀的 Android 工程，暂定以下两个

    • iosched https://github.com/google/iosched
